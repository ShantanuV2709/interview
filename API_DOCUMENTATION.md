# API Endpoint Documentation (FastAPI)

This document outlines the REST and WebSocket endpoints provided by the **FastAPI Application** (`app.py`). This is the modern service layer for the Sarvam Interview Demo.

---

## 1. REST API (`app.py`)
- **Port**: 3000 (standard)
- **Base URL**: `http://localhost:3000/api/v1`
- **Interactive Docs**: [http://localhost:3000/docs](http://localhost:3000/docs)

### Endpoints

#### [POST] `/generate-questions`
- **Description**: Generates 6 interview questions based on a job description.
- **Payload**: `{"job_title": "...", "job_description": "...", "experience": "..."}`
- **Response**: JSON object containing an array of questions with categories and keywords.

#### [POST] `/stt`
- **Description**: Transcribes a raw audio buffer using Deepgram nova-2.
- **Payload**: Raw binary audio (default `audio/webm`).
- **Response**: `{"transcript": "..."}`

#### [POST] `/tts`
- **Description**: Generates audio from text using Sarvam bulbul:v2.
- **Payload**: `{"text": "..."}`
- **Response**: Binary MP3 audio stream.

#### [GET] `/tts`
- **Description**: Utility endpoint for browser-based TTS testing.
- **Query Param**: `text` (e.g., `?text=hello`)
- **Response**: Direct MP3 audio playback.

#### [POST] `/llm-turn`
- **Description**: Fetches a single non-streaming conversational response from the Emma persona.
- **Payload**: `{"prev_context": "...", "transcript": "...", "next_question": "..."}`
- **Response**: `{"response": "...", "action": "next|end|repeat|previous"}`

#### [POST] `/score`
- **Description**: Evaluates the entire interview transcript.
- **Payload**: `{"title": "...", "experience": "...", "transcripts": [...]}`
- **Response**: Detailed JSON nested scores, feedback, strengths, and recommendations.

## 2. Unified WebSocket Stream (`app.py`)
- **Endpoint**: `ws://localhost:3000/ws/v1/interview-stream`

### Protocol
The unified stream handles the complex orchestration of LLM tokens and concurrent TTS audio generation.

- **Message Interface**:
  - **Action: `ask`**: Sends the current turn context (`prev`, `transcript`, `nextQ`).
- **Binary Responses**: The server streams back binary audio chunks (MP3) as they are generated by Sarvam.
- **JSON Messages**:
  - `{"type": "token", "text": "..."}`: Real-time LLM tokens for UI rendering.
  - `{"type": "control", "action": "done"}`: Signals the end of the current turn.
